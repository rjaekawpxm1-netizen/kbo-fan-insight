# 데이터 분석 → 앱 개선 연결

> 본 프로젝트의 분석 결과를 KBO 앱 개선 가설과 기능 요구사항으로 연결한다.
> 이 문서가 "데이터만 다루는 지원자"와 "앱을 개선할 줄 아는 지원자"를 가르는 핵심 산출물이다.

## [결론]

수집·분석한 세 축(관중수 예측 / 팬 감성 / 콘텐츠 이용현황)은 각각 KBO 앱의 구체적 개선 포인트로 연결된다. 분석은 그 자체가 목적이 아니라 **앱에서 무엇을 바꿀지 판단하기 위한 근거**다.

## [분석 → 가설 → 기능]

| 분석 발견(예시) | 앱 개선 가설 | 연결 기능(요구사항) |
|---|---|---|
| 평일·특정 구장 관중 급감 | 평일 경기 관심 유도 필요 | 평일 한정 추천 경기 카드·혜택 푸시 |
| 요일·상대팀에 따른 관중 편차 | "오늘 볼 만한 경기" 개인화 | 응원팀 기준 추천 경기 노출 |
| 팀별 댓글 부정 비율 급증 시점 존재 | 이슈 발생 시 대응 콘텐츠 필요 | (운영자용) 팀 감성 추이 알림 |
| 하이라이트 조회가 인터뷰 대비 우세 | 콘텐츠 노출 우선순위 조정 | 홈 탭 콘텐츠 추천 정렬 개선 |

> ⚠ 위 발견은 분석 파이프라인이 산출하는 지표의 **예시**다. 실데이터 수집 후 실제 수치로 갱신해야 한다(예: 구장별 평일 관중 평균, 팀별 부정 비율).

## [근거가 되는 분석 산출물]
- **관중수 예측**(XGBoost, 시간순 검증): 요일·구장·상대팀 효과를 정량화 → 추천 경기 로직 근거
- **팬 감성분석**(하이라이트 댓글): 팀별·시점별 반응 → 콘텐츠/커뮤니케이션 우선순위
- **콘텐츠 이용현황**: 영상별 반응량 → 홈피드 정렬 근거

## [실무 적용 문구(보고서/PPT용)]
- "관중수 예측 결과를 앱의 '오늘의 추천 경기' 카드 노출 로직 근거로 활용"
- "팀별 감성 추이를 운영자 대시보드 알림으로 연결해 이슈 대응 리드타임 단축"
- "콘텐츠 이용현황 기반으로 홈 탭 추천 정렬을 데이터 기반으로 전환"

## [검증 포인트]
- 예시 발견을 실데이터 수치로 교체하기 전에는 "가설"로 표기.
- 운영자용/사용자용 기능을 구분(감성 알림은 운영자용).
- 기능화 전 개발·기획 부서와 우선순위·실현가능성 협의 필요(공고의 "유관 부서 협업 자료" 항목과 연결).
