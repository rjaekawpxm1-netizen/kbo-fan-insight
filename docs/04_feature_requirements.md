# KBO 앱 기능 요구사항 정의서 (초안)

> 벤치마킹(docs/02)과 분석→개선 연결(docs/05)을 종합한 기능 요구사항 초안.
> 공고 항목 "앱 개선사항 정리 및 기능 요구사항 문서화"에 대응. 우선순위는 제안값(협의 전).

## [결론]

최우선은 **응원팀 기반 개인화 홈(F-01)**이다. 진입 빈도가 가장 높고, 이후 추천·기록·참여 기능의 토대가 된다. 다음으로 개인 관람 기록(F-03), 추천 경기(F-02) 순으로 권장한다.

## [요구사항 목록]

| ID | 기능명 | 설명 | 우선순위 | 데이터·분석 근거 |
|---|---|---|---|---|
| F-01 | 응원팀 기반 개인화 홈 | 팀 선택 시 그 팀 일정·기록·콘텐츠 중심으로 홈 재배치 | 상 | 벤치마킹(MLB 홈피드), 콜드스타트 우회 |
| F-02 | 추천 경기 카드 | "오늘 볼 만한 경기"를 개인화 노출 | 상 | 관중수 예측, 콘텐츠 이용현황 |
| F-03 | 개인 관람 기록 | 직관/집관 기록·개인 통계(승요력 등) | 상 | 벤치마킹(오늘의 야구), 리텐션 |
| F-04 | 콘텐츠 추천 정렬 | 홈 탭 콘텐츠를 이용현황 기반 정렬 | 중 | 콘텐츠 이용현황 분석 |
| F-05 | 팬 참여(예측·포인트) | 승부 예측·출석·퀴즈형 포인트 | 중 | 벤치마킹(프로야구 LIVE) |
| F-06 | 팀 감성 추이 알림(운영자) | 팀별 부정 반응 급증 시 운영자 알림 | 중 | 감성분석 |
| F-07 | 콘텐츠 태깅 체계 | 팀·선수·상황 태그 부여(추천 기반 데이터) | 중 | 벤치마킹(Netflix 태깅) |
| F-08 | 기존 AI 챗봇 고도화 | 앱에 **이미 있는 AI 챗봇**(docs/01 확인)을 응원팀 기반 개인화 질의·콘텐츠 추천으로 확장 | 중 | docs/01(챗봇 기존재), LLM 팬 반응 요약 |

> ⚠ AI 기능은 **신규 도입이 아니라 기존 챗봇 확장**이다. docs/01에서 KBO 앱에 AI 챗봇이 이미 존재함을 확인했으므로, "새 AI 비서 추가"가 아니라 기존 자산 고도화로 접근한다.

> 우선순위 기준: 사용 빈도·구현 난이도·데이터 가용성. 확정값 아님(개발·기획 협의 필요).

## [비기능/제약]
- 개인화 데이터 활용은 개인정보 동의·보관 정책 검토 필요.
- 영상 콘텐츠는 중계권(티빙 등) 제약 확인 필요.
- 추천 로직은 콜드스타트(신규/저활동 사용자) 대응 설계 필요.

## [검증 포인트]
- KBO 공식 앱의 현재 기능과 중복/공백을 실제 앱 캡처로 대조해야 우선순위 확정 가능(docs/01 연계).
- 각 기능의 데이터 가용성(실데이터 vs 합성) 재확인.
- 우선순위·범위는 유관 부서 협의 후 확정.
