# AI 개인화 서비스 사례 조사 + KBO 적용 제안

> KBO 미디어사업 공고 명시 항목 "AI 개인화 서비스 사례 조사"에 대응. 타 산업 검증 사례를
> 조사하고 KBO 앱 적용 방향을 제안한다. 직접 구축이 아니라 사례 기반 제안임을 명확히 한다.

## [결론]

개인화 추천은 OTT·커머스에서 이미 매출·체류의 핵심 동력으로 검증됐다. KBO 앱은 이를 **응원팀 기반 콘텐츠 추천 + 행동로그 기반 홈피드**로 차용하되, 콜드스타트(신규 사용자 정보 부족) 문제를 응원팀 선택으로 우회하는 설계가 현실적이다.

## [조사 사례]

| 서비스 | 방식 | 핵심 효과 | KBO 시사점 |
|---|---|---|---|
| Netflix | 콘텐츠 상세 태깅 + 협업/콘텐츠 기반 하이브리드 | 콘텐츠 소비의 다수가 추천에서 발생 | 영상·기록 콘텐츠 태깅 체계 필요 |
| Amazon | 아이템 기반 협업 필터링 | 매출 상당 비중이 추천 기여 | "이 선수 좋아하면 이 콘텐츠" 연관 추천 |
| 무신사·네이버쇼핑 | 클릭·검색·장바구니·구매 등 행동로그 종합 | 재방문·구매전환 상승 | 앱 행동로그(시청·클릭) 기반 홈 재배치 |

### 사례 요지(조사 내용)
- **Netflix**: 콘텐츠를 상세 태그로 분류해 정교한 추천을 구현하며, 전체 콘텐츠 소비의 약 4분의 3이 추천 알고리즘을 통해 발생하는 것으로 알려져 있다. 협업 필터링과 콘텐츠 기반을 결합한 하이브리드 방식이 핵심이다.
- **Amazon**: 아이템 기반 협업 필터링("이 상품을 본 사람이 함께 본 상품")으로, 매출의 상당 비중이 추천에서 발생하는 것으로 보고된다.
- **무신사·네이버쇼핑**: 클릭·검색·장바구니·구매 이력 등 사용자 행동 데이터를 종합 분석해 맞춤 상품을 제시하고, 푸시·이메일 등 다채널로 개인화를 확장한다.

> 추천 방식 분류: **협업 필터링**(유사 사용자/아이템 기반), **콘텐츠 기반**(아이템 속성 기반), 둘을 합친 **하이브리드**(콜드스타트 완화).

## [KBO 앱 적용 제안]

> 전제: KBO 앱에는 **AI 챗봇이 이미 존재**(docs/01 확인). 따라서 아래 개인화는 신규 AI 도입이 아니라 **기존 챗봇·홈을 개인화·추천으로 확장**하는 방향이다.

1. **콜드스타트 우회** — 가입 시 응원팀 선택을 받아 초기 개인화의 출발점으로 사용(행동 데이터가 쌓이기 전까지의 공백 해소).
2. **행동로그 기반 홈피드** — 영상 시청·기록 조회·클릭 로그를 종합해 홈 콘텐츠를 재정렬(무신사·네이버쇼핑 방식).
3. **콘텐츠 태깅 체계** — 영상·기록 콘텐츠에 팀·선수·상황(역전·끝내기 등) 태그를 부여해 콘텐츠 기반 추천 가능하게(Netflix 방식).
4. **연관 추천** — "이 선수 콘텐츠를 본 팬이 함께 본 콘텐츠"(아마존 아이템 기반 방식).

## [데이터 분석과의 연결]
- 팬 세그먼트/콘텐츠 이용현황 분석 → 추천 입력 피처
- 감성분석 → 부정 반응 콘텐츠 노출 조정
- 관중수 예측 → "오늘의 추천 경기" 개인화 카드

## [검증 포인트]
- 인용 수치(추천 기여 비중 등)는 출처별·시점별로 차이가 있어 **제안 근거 수준**으로만 사용(정밀 수치는 원 출처 재확인).
- KBO는 행동로그가 공개되지 않아, 본 프로젝트의 로그는 합성 설계 샘플임(실서비스 적용 시 실데이터 필요).
- 개인정보·동의 등 개인화 데이터 활용의 법적 요건은 별도 검토 필요.
